# Projects

Active work with a defined outcome and a deadline (even a soft one). Each project gets its own folder containing notes, drafts, deliverables, and a `_index.md`.

## What counts as a project

- Has a **clear outcome** (something will exist or change)
- Has **multiple steps** (otherwise it's a task)
- Has an **end** (otherwise it's an area)

If something fails any of these tests, it might belong in [[05_Knowledge/README]] (durable reference), [[06_Memory/README]] (something that happened), or just be a one-off task.

## Lifecycle

1. **Start:** create folder, copy template, define outcome and first three steps.
2. **Run:** keep notes, drafts, and deliverables inside the folder. Update `_index.md` weekly.
3. **Decide endings:** when blocked, mark `paused`. When done, mark `done` and run the close-out.
4. **Close out:** capture key decisions to `06_Memory/decisions/`, learnings to `06_Memory/learnings/`. Move folder to `04_Projects/archive/`.
