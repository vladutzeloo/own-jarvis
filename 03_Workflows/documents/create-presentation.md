# Create a Presentation

Slide deck for a meeting, pitch, or review.

## Trigger

- "Make a deck about X"
- "Slides for the X meeting"
- "Pitch deck for Y"

## Steps

**1. Read the pptx skill.** Always. Invoke [[02_Capabilities/skills/pptx]].

**2. Clarify the meeting context.** Who's the audience? How long is the slot? What's the decision or action you want from them? The deck serves the meeting, not the other way around.

**3. Outline the narrative arc.** Even a 5-slide deck has a beginning, middle, end. Most arcs:
   - Pitch: problem → solution → market → product → traction → team → ask
   - Update: what we said we'd do → what we did → what we learned → what's next
   - Review: context → what's working → what's not → decisions needed
   - Recommendation: question → options → tradeoffs → recommendation → next steps

**4. One idea per slide.** If a slide has two ideas, split it.

**5. Speaker notes for anything that won't be on the slide.** The slide is for the audience; the notes are for the presenter.

**6. Save it.** Project folder if project-bound, vault root otherwise.

## Verification step

Read every slide title in order — they should tell a story by themselves. If the titles read like a coherent paragraph, the deck has a spine. If not, restructure.
