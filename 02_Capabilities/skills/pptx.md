# Skill: pptx

Create, read, and edit PowerPoint presentations and slide decks.

## Triggers

- Any mention of "deck", "slides", "presentation", ".pptx", "pitch deck"
- Reading or extracting text from a .pptx (even if extracted content goes elsewhere)
- Working with templates, layouts, speaker notes, or comments
- Combining or splitting slide files

## When to use

Whenever a .pptx file is involved — input, output, or both. If Vladimir says "summarize my deck", use this skill to read it first.

## When NOT to use

- The deliverable is a Word doc with images → use [[02_Capabilities/skills/docx]]
- A simple visual answer in chat → use HTML/SVG artifacts instead

## Workflow

1. Skill auto-loads on trigger, or invoke via `Skill` tool with `skill: "pptx"`.
2. Read SKILL.md before touching any slides.
3. Plan the deck structure first — title, agenda, sections, conclusion. Don't start at slide 1.
4. Apply the skill's layout and font guidance consistently across slides.
5. Save to `C:\Users\vdzoo\Documents\obsidian\brain\` or project folder.

## Common patterns

- **Pitch deck:** problem → solution → market → product → traction → team → ask. ~10-15 slides.
- **Executive summary:** title, one-page summary, key findings (3-5 slides), recommendation, appendix.
- **Internal review:** context, what's working, what's not, decisions needed.
