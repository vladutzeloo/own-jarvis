# Knowledge

Durable subject knowledge. Things that are true regardless of any specific project.

The split that matters: knowledge here ages on the scale of a field (years to decades). [[06_Memory/README]] ages on the scale of a life (specific events, decisions, episodes).

## Subfolders

- **`domains/`** — knowledge organized by subject area. Create a folder per domain (`domains/finance/`, `domains/legal/`, `domains/programming/`). Inside, use a MOC file per topic.
- **`references/`** — links and notes on external resources. Books, papers, tools, websites worth remembering.

## How to add knowledge

1. Pick the right subfolder. If a new domain is needed, create the folder + a `README.md` MOC.
2. One concept per file. If a note crosses two concepts, split it.
3. Use the `domain/<name>` tag in addition to filing it in the right folder.
4. Cross-link aggressively — knowledge that doesn't connect to other knowledge is hard to find later.

## Anti-patterns

- **Filing project-specific notes here.** They belong in `04_Projects/<project>/`.
- **Filing transient news.** Put it in `06_Memory/episodes/` if it matters; trash it if it doesn't.
- **Lone unfiled notes.** Every note should have at least one wikilink in or out.
