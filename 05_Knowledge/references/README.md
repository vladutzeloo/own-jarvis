# References

External resources worth remembering — books, papers, tools, websites.

## Naming

`<short-title>.md`, with frontmatter:

```yaml
---
title: Full Title
type: knowledge
tags: [reference, book|paper|tool|site, domain/<domain>]
source: URL or citation
updated: YYYY-MM-DD
---
```

## Pattern per file

For each reference, keep it short:
- Why it matters (one paragraph)
- Three to five takeaways
- Related notes/projects

A reference file is not a book report. If a single takeaway gets long, promote it to a knowledge note in `domains/`.

> *Empty until you add references.*
