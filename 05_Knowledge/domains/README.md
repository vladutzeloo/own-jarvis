# Domains

One folder per subject area. Create a domain folder when knowledge in that area starts overflowing into ad-hoc files.

## Domains

> *Empty until you add one. Standard pattern:*
>
> ```
> domains/<domain>/
>   README.md        # MOC for this domain
>   <topic>.md       # individual notes
> ```

## Suggested first domains

Don't pre-create — wait until there's actual content. Common starting points for many people:

- `programming/` — languages, frameworks, patterns
- `finance/` — investing, taxes, planning
- `health/` — fitness, nutrition, sleep, medical
- `business/` — strategy, operations, management
